## 1. 设计简报

- 产品定位：面向高频锁屏场景的轻量提醒工具，不追求复杂任务管理，而强调“最重要的事稳定看见”。
- 目标用户：
  - 容易忘事、但不愿维护重型待办系统的人
  - 频繁查看锁屏、依赖视觉提醒的人
  - 需要在“可见性”和“隐私性”之间做平衡的人
- 使用时刻：
  - 出门前、会议前、取快递前等短时高压力时刻
  - 用户只愿花几秒钟完成记录、确认与回看
- 核心承诺：
  - 重要事项一眼可见
  - 快速添加不打断节奏
  - 隐私事项可安全降级展示
- 业务目标：
  - 提高首次创建与首次启用锁屏展示的转化率
  - 提高用户日常回看频率与事项完成率
  - 通过可信的隐私保护降低卸载与流失

## 2. 平台基线

- 平台行为基线采用 iOS HIG。
- 导航、返回、可点击区域、开关控件、危险操作反馈、权限说明与无障碍对比度必须遵守 iOS 基线，不因风格化处理而破坏习惯路径。
- 视觉风格允许脱离系统默认外观，但交互心智不允许脱离系统默认预期。

## 3. 当前方向诊断

- 当前仓库已有 `app.pen`、Flutter 页面与多张预览图，但此前缺少一个正式的全局设计包来约束后续 Pencil 与 Flutter 不再各自解释。
- `style-01` 的价值不只是“更好看”，而是同时给出了：
  - 锁屏展示的核心产品时刻
  - 首页、历史页、编辑页与隐私页的统一物料语言
  - 可被组件化的纸张、图钉、暖白背景、橄榄绿色 CTA 与轻纸质纹理
- 若继续只把这些图当作灵感而不是设计源，后续实现很容易出现：
  - 页面层级被还原成默认表单页
  - 锁屏卡片与应用内卡片语言分裂
  - 隐私页、历史页、空态页缺少同一视觉节奏

## 4. 方向对比与推荐

### 4.1 推荐方向

- 采用 `style-01 warm note` 作为全局设计参考基线。
- 设计姿态：安静、温和、可信、带轻微手作触感，但仍然是效率工具，不是生活方式海报。
- 视觉记忆点：
  - 便签纸与图钉的轻拟物化表达
  - 暖米白纸面、低饱和橄榄绿主色、克制的陶土色提醒点
  - 留白充分、内容不拥挤、视觉焦点始终围绕“当前最重要事项”

### 4.2 不作为主基线的方向

- `style-02 night focus`
  - 优点：隐私、安全与夜间专注氛围很强。
  - 不选为主基线原因：整体情绪过重，更适合作为深色主题与隐私场景的局部语言，而不是全天候默认基线。
- `style-03 editorial system`
  - 优点：排版成熟、品牌记忆强、页面信息清晰。
  - 不选为主基线原因：偏编辑感海报，任务卡的“便签持续存在感”弱于 `style-01`，不如 `style-01` 贴合“屏幕便签”心智。

## 5. 设计冻结卡

### 5.1 商业目标

- 用最低录入成本让用户养成“出门前看一眼锁屏便签”的日常习惯。
- 把“不会忘”和“能安心展示”作为 retention 核心，而不是把功能复杂度做高。

### 5.2 目标用户与场景

- 通勤前要确认证件、快递、合同、会议资料的人。
- 在办公室、地铁、电梯等高分心场景里，只能接受极短交互的人。
- 对提醒有需求，但不希望锁屏泄露完整内容的人。

### 5.3 核心路径

1. 用户从首页或快速添加入口输入一句提醒。
2. 用户可补充提醒时间、重复、展示时长与隐私模式。
3. 保存后，事项以便签卡形式出现在首页与锁屏展示快照中。
4. 用户通过首页、历史页或详情页完成、恢复或删除事项。
5. 用户在下一次查看锁屏时继续被最重要事项提醒，形成回看闭环。

### 5.4 页面范围

- 首页任务列表
- 快速添加弹层 / 快速添加页
- 事项详情 / 编辑页
- 最近完成页
- 最近删除页
- 设置页
- Widget 展示设置页
- 隐私设置页
- 锁屏快照 / Widget 快照预览

### 5.5 艺术指导

- 产品姿态：安静助手，不说教，不科技炫耀，不做重生产力仪表盘。
- 信息密度：偏稀疏，优先一眼读懂，再考虑次级信息。
- 视觉温度：暖静、纸感、低刺激。
- 去原生策略：
  - 主要依赖排版节奏与材质层次去掉默认系统设置页既视感
  - 不依赖大面积插画、重阴影或泛滥渐变
- 主要抛光杠杆：
  - `hierarchy`：让“当前最重要事项”成为绝对焦点
  - `material`：用纸面、卡片、图钉和轻阴影建立可信的实体感

### 5.6 信息层级

- 主导区：当前最重要或最近到期的便签卡。
- 次级区：提醒时间、重复、隐私开关、完成状态等控制信息。
- 支撑区：历史入口、设置入口、说明文案与预览。
- 主操作：
  - 首页主操作是“新增提醒”
  - 编辑页主操作是“完成 / 保存便签”
  - 删除、恢复等破坏性行为必须降级到次级层

### 5.7 视觉系统

- 字体：
  - 标题与核心便签文案采用高对比、略带纸面书写气质的衬线/类书写表达
  - 次级信息与功能标签保持现代无衬线，避免整体过于复古
- 间距：
  - 页面外边距宽于系统默认表单页
  - 卡片内边距偏大，让一句提醒在小屏上也显得从容
- 色彩角色：
  - 暖米白作为背景与大面基底
  - 橄榄绿承载主按钮、开关选中态、图钉与关键强调
  - 陶土橙承载时间提醒、恢复操作与次级强调
- 表面层级：
  - 页面背景最轻
  - 纸面卡片高于背景
  - 弹层 / 底部面板高于纸面卡片
- 圆角与边界：
  - 卡片圆角偏柔和但不过度圆润
  - 大多数卡片弱化边框，更多依赖层次与阴影
- 图标姿态：
  - 线性、小而克制，避免厚重拟物图标
- 动效角色：
  - 仅用于卡片出现、保存反馈、开关切换与快照预览过渡
  - 不允许炫技式位移动效

### 5.8 状态矩阵

| 状态 | 必须覆盖页面 | 设计表达 |
| --- | --- | --- |
| ideal | 首页、锁屏快照、详情页 | 主便签卡完整展示，一眼看清当前任务与提醒时间 |
| empty | 首页、最近完成、最近删除 | 保留纸面感空态，但文字要更轻，不可伪装成空白错误 |
| loading | 首页、历史页、设置预览 | 使用轻骨架或低对比占位块，不出现跳闪 |
| error | 快照刷新失败、保存失败、通知失败 | 采用轻提示条或内联说明，不把整页染成错误态 |
| permission | 通知权限、Widget 能力、隐私授权说明 | 用解释型卡片和非阻断按钮说明影响，主链路仍可继续 |
| partial_data | Widget 刷新失败、历史仅部分加载 | 明确“保留最后一次可用快照”，不要让用户误以为任务丢失 |
| disabled | 保存按钮、完成按钮、重复项控件 | 使用低对比暖灰，不可与普通次级按钮混淆 |
| success | 保存成功、恢复成功、完成成功 | 使用短暂轻反馈，维持安静气质，不做大面积庆祝态 |
| premium_or_locked | 当前版本不适用 | 标记 `not_applicable`，但保留结构以便未来 Pro 功能扩展 |

### 5.9 商业化触点

- 当前 P0 不引入显式订阅或升级墙。
- 当前最重要的商业时刻是“首次看到锁屏便签确实帮我记住事情”，这是信任与留存触点，不是付费触点。

### 5.10 必须一致项

- 纸面卡片始终是最核心的组件隐喻。
- 主按钮、开关选中态与关键确认动作必须使用同一橄榄绿色角色。
- 便签正文必须始终保持最高阅读优先级，时间、重复、隐私说明只能退居第二层。
- 隐私模式不能改变产品主语义，只能改变展示密度与内容暴露级别。
- 锁屏快照和应用内卡片必须共用同一视觉家族，不能做成两套无关设计。

### 5.11 允许工程化调整项

- 为了适配小屏、动态字体和不同系统字号，允许微调：
  - 卡片内边距
  - 标题字号档位
  - 弹层高度与底部安全区留白
- 为了适配 Flutter 与 WidgetKit 的实现限制，允许弱化：
  - 纸张细纹理颗粒
  - 过于细小的阴影层
  - 装饰性背景光斑
- 不允许工程侧自行修改：
  - 主色角色
  - 主次层级
  - 便签卡的核心比例与留白逻辑

### 5.12 平台不可偏离项

- 返回、关闭、删除、恢复、开关切换都必须符合 iOS 触达与反馈习惯。
- 破坏性行为必须二次确认或至少具备明确上下文，不允许把“删除”藏进主按钮样式。
- 文本、按钮、图标和触达区必须满足基础可访问性对比度与点击面积。

### 5.13 Prompt 约束

- 继续生成预览、Pencil 或 Flutter 设计时，必须保留：
  - 暖纸感便签
  - 橄榄绿主操作
  - 轻陶土提醒色
  - 高留白、低密度、强主焦点
- 禁止转向：
  - 玻璃拟态
  - 科技霓虹
  - 仪表盘式高密度卡片墙
  - 过度插画化、过度摄影化背景

### 5.14 交付验收门

- 3 秒内能判断“当前最重要事项是什么、是否设置提醒、怎样继续添加”。
- 首页、历史页、详情页、隐私页必须属于同一视觉系统，而不是四种孤立样式。
- 锁屏快照与应用内便签必须能被识别为同一组件家族。
- 状态矩阵不能只做 happy path；至少要补齐 empty、permission、partial_data、disabled。
- Flutter 落地后仍需允许动态字体、长文案与权限降级，不得为了还原视觉而牺牲可用性。

## 6. Flutter 实现守则

- 应保留：
  - 主次层级
  - 卡片比例
  - 暖白背景与橄榄绿主色的角色分工
  - 隐私降级展示原则
- 不应机械照抄：
  - 背景纸纹颗粒
  - 光影摄影化细节
  - 所有 mockup 中的装饰性摆件与场景光影
- Widget、通知、小屏锁屏等弱能力场景优先保留“层级、颜色角色、隐私策略”，其次才是材质细节。
