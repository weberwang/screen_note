## 1. 设计简报

- 设计读取：
  - 这是一个面向个人高频锁屏场景的移动效率工具，服务对象是希望在极短时间内记录并持续看见重要事项的人，整体应当呈现安静、可信、克制、反模板的高质感移动产品语言。
- 产品核心承诺：
  - 最重要事项一眼可见
  - 记录动作足够短
  - 隐私暴露可控
- 业务目标：
  - 提高首次创建与首次启用锁屏展示的转化
  - 提高日常回看频率和事项完成率
  - 用可信的隐私降级降低卸载和流失
- 目标用户：
  - 容易忘事但拒绝重型待办系统的人
  - 高频看锁屏、依赖视觉提示的人
  - 需要在提醒可见性与隐私之间取得平衡的人
- 关键使用时刻：
  - 出门前确认证件、快递、资料
  - 会议前和通勤中处理临时提醒
  - 睡前记录次日必须完成的一件事

## 2. 平台基线

- 平台行为基线采用 iOS HIG。
- 交互、点击热区、危险操作、权限说明、可访问性和反馈节奏必须服从 iOS 默认认知，不得为了风格破坏可用性。
- 视觉可以去模板化，但不能抽象到让用户需要重新学习系统逻辑。
- 锁屏、小组件、通知和系统快捷入口必须默认遵守隐私优先原则。

## 3. Taste 方向

### 3.1 来源与参数

- 视觉方向完全依赖 `design-taste-frontend` 的 anti-slop 约束。
- 方向参数：
  - `DESIGN_VARIANCE: 6`
  - `MOTION_INTENSITY: 3`
  - `VISUAL_DENSITY: 3`

### 3.2 方向解释

- 不做系统设置页模板，也不做视觉炫技海报，因此采用中等偏上的结构变化，但不走过度艺术化路线。
- 动效只承担层级切换、结果反馈和预览变化，不承担品牌叙事表演。
- 信息密度保持偏低，确保“当前最重要事项”在三秒内绝对成为第一焦点。

### 3.3 反模板约束

- 禁止泛紫渐变、通用 SaaS Hero 套路、无意义玻璃拟态、三等分卡片墙。
- 禁止大面积装饰性摄影或插画遮蔽主任务。
- 禁止把主 CTA 埋进一组同权重按钮中。
- 禁止让页面回退成白底表单列表风格。

### 3.4 预览图归纳出的视觉方向

- 首页确定了“暖纸背景 + 深橄榄主卡片 + 陶土强调动作”的主视觉轴。
- 历史页确定了“纸边分隔 + 归档卡片 + 小票/手写感缩略图”的次级列表语言。
- 详情 / 隐私页确定了“上半部信息确认 + 中段锁屏预览 + 底部强主 CTA”的单任务完成结构。
- 三张图共同确认：
  - 标题区采用高识别度衬线标题
  - 功能文字保持现代无衬线
  - 纸面材质只作温和背景，不可盖过信息

## 4. 商业目标

- 让用户形成“看锁屏就能想起最重要事项”的稳定习惯。
- 让“记录一条提醒”成为无负担动作，而不是另一个管理负担。
- 让隐私模式成为信任基建，而不是付费点或附属设置。
- 让产品在第一周内建立“不会漏、不会乱、不会泄露”的稳定认知。

## 5. 目标用户与场景

- 用户一：
  - 忘事型用户，主要痛点不是任务多，而是记录后仍然忘记看。
- 用户二：
  - 轻量待办用户，不愿维护标签、分组、项目树。
- 用户三：
  - 高频锁屏用户，希望在抬手瞬间看到最重要提醒。
- 用户四：
  - 隐私敏感用户，希望别人看见设备时仍不暴露正文。

## 6. 核心路径

1. 用户从首页或快捷入口输入一句提醒。
2. 用户可补充提醒时间、重复规则和隐私模式。
3. 保存后，事项立即成为首页主焦点，并准备进入锁屏或小组件快照。
4. 用户可在首页、详情页、历史页完成、恢复或删除事项。
5. 用户在下次查看锁屏时继续被最重要事项提示，形成回看闭环。

## 7. 页面范围

- 首页任务列表
- 快速添加弹层或快速添加页
- 事项详情 / 编辑页
- 最近完成页
- 最近删除页
- 设置页
- 锁屏 / Widget 展示设置页
- 隐私设置页
- 锁屏 / Widget 预览页

## 8. 艺术指导

### 8.1 产品姿态

- 安静、可信、克制。
- 不做仪表盘，不做科技霓虹，不做营销海报式夸张叙事。
- 不靠复杂元素制造“高级感”，而是靠层级、节奏、留白与材质暗示建立品质。

### 8.2 视觉温度

- 偏温和、偏生活化、偏可信。
- 颜色角色要少而稳定，不允许多主色并存。
- “提醒感”要可见，但不制造焦虑。

### 8.3 去模板化策略

- 用更强的层级与节奏替代系统默认表单页结构。
- 用材质感、信息编排和主焦点构图替代花哨装饰。
- 用统一的共享组件家族替代每个页面单独做样式。

## 9. 信息层级

- 第一阅读层：
  - 当前最重要事项正文
- 第二阅读层：
  - 时间、重复规则、隐私说明、状态标签
- 第三阅读层：
  - 历史入口、说明文案、辅助操作、系统能力解释
- 主 CTA：
  - 首页主 CTA 永远是“新增提醒”
  - 编辑页主 CTA 永远是“保存”或“完成”
- 次级操作：
  - 删除、恢复、更多操作必须降级显示，不能与主 CTA 同权重竞争

## 10. 视觉系统

### 10.1 排版

- 标题和主事项正文必须形成明确阅读梯度。
- 正文优先级高于标签、提示、说明。
- 页面不允许因为“简洁”而把所有信息压成同字号和同色块。
- 大标题使用带书卷感的衬线风格，功能正文与说明保持现代无衬线。

### 10.2 色彩

- 整体采用低密度、低噪声、单主轴配色。
- 主色只承担确认、主 CTA、选中状态和结构性强调。
- 警示色和提醒色只承担到期、恢复、风险提示等语义，不得抢主 CTA。
- 禁止出现新的高饱和第二主色。

### 10.3 间距与节奏

- 页面顶部必须有呼吸感。
- 主事项容器必须拥有明显大于普通列表项的留白。
- 历史页和设置页也必须遵守同一垂直节奏，不允许退化成挤压型列表。

### 10.4 表面与材质

- 允许存在温和的纸面材质、撕边和弱阴影暗示，但不得让装饰主导信息。
- 卡片、面板和预览容器的层级要清楚，但不过度堆叠阴影。
- 共享组件的质感优先于装饰细节。

### 10.5 动效

- 仅允许：
  - 卡片进入
  - 保存反馈
  - 开关切换
  - 预览变化过渡
- 禁止：
  - 大幅位移动画
  - 滚动炫技
  - 影响阅读的持续动效

## 11. 状态矩阵

| 状态 | 必须覆盖页面 | 设计意图 |
| --- | --- | --- |
| ideal | 首页、详情、锁屏预览 | 一眼确认当前最重要事项 |
| empty | 首页、最近完成、最近删除 | 强调下一步动作，不制造失败感 |
| loading | 首页、历史、设置预览 | 低干扰占位，不跳闪 |
| error | 保存失败、快照刷新失败、通知失败 | 解释失败边界，不制造恐慌 |
| permission | 通知权限、Widget 能力、隐私说明 | 明确能力降级，不阻断主链路 |
| partial_data | Widget 刷新失败、历史部分加载 | 强调“数据仍在，只是展示降级” |
| disabled | 保存按钮、完成按钮、局部开关 | 清楚表达不可用，但仍可识别 |
| success | 保存成功、恢复成功、完成成功 | 短促确认，不做庆祝动画 |
| premium_or_locked | 当前阶段 | `not_applicable` |

## 12. 共享组件与约束

### 12.1 共享组件族

- 主事项卡片
- 快速录入条
- 历史事项行
- 主 CTA 按钮
- 次级面板
- 设置开关行
- 隐私预览卡
- 过滤分段按钮

### 12.2 必须一致项

- 主事项正文必须始终是第一阅读层。
- 锁屏、小组件、应用内主卡片必须属于同一视觉家族。
- 隐私模式只能降级内容暴露，不得改变主层级和主任务提示逻辑。
- 首页主 CTA 必须持续保持“新增提醒”为主操作。
- 共享组件不能在不同页面被重新解释成不同设计系统。

### 12.3 允许工程化调整

- 可微调卡片内边距、字号档位、底部安全区留白。
- 可弱化过细纹理、极细阴影和装饰性背景细节。
- 不可擅自改动主色角色、层级顺序、主事项卡片比例和隐私降级原则。

## 13. 平台不可偏离项

- 返回、关闭、删除、恢复、开关切换都必须符合 iOS 触达与反馈预期。
- 危险操作必须拥有明确上下文，不能伪装成主 CTA。
- 锁屏、小组件、通知和系统快捷入口不得泄露私密事项正文。
- 动态字体、长文案和权限降级必须在设计上被预留，而不是实现阶段临时打补丁。

## 14. Prompt 约束

- 后续若继续生成视觉草案或补充静态设计证据，必须继续继承：
  - 安静、可信、克制的产品姿态
  - 单主轴配色和低密度信息节奏
  - 主事项正文绝对优先的层级原则
  - 主 CTA 清晰、次级操作降级
  - 隐私模式只降级暴露，不改写任务结构
- 后续生成时禁止转向：
  - 默认系统设置页风格
  - 霓虹科技感
  - 玻璃拟态泛滥
  - 多主色
  - 数据仪表盘式高密度排布

## 15. 交付验收门

- 用户必须能在三秒内理解：
  - 当前最重要事项是什么
  - 是否有提醒或时间约束
  - 下一步如何新增或确认
- 首页、历史页、详情页、隐私页必须属于同一视觉系统，而不是各自独立样式。
- 锁屏快照与应用内主卡片必须可被识别为同一组件家族。
- 状态矩阵不能只覆盖 happy path，至少要完整覆盖 empty、permission、partial_data、disabled。
- 当前产物已具备冻结向静态视觉证据，可进入共享冻结门禁和后续模块拆分。
