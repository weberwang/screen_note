# history-center 空状态重做设计

## 1. 目标

把 `history-center` 在“完全没有历史记录”时的显示层，重做为基于最新效果图的全页空状态表达，不再沿用当前卡片式空态。

本次成功标准：

- 仅在 `snapshot.isEmpty` 时启用新空状态布局。
- 新空状态整体贴近效果图：顶部 `History` 标题、居中插画区、主标题、说明文案、底部圆形 `+` 动作。
- `+` 动作点击后直接进入现有的新建事项编辑链路。
- 有历史记录时的最近完成 / 最近删除列表不受影响。

## 2. 范围

### 范围内

- `history-center` 空状态分支的页面结构
- 空状态专用展示组件
- 空状态 `+` 按钮交互
- 对应页面测试更新

### 范围外

- 最近完成 / 最近删除分区的正常态列表
- 恢复用例、Provider、仓储和 `task-flow` 真源
- 路由结构调整
- 生成新的效果图

## 3. 设计决策

### 3.1 空状态采用独立分支布局

不在现有列表内容流里硬塞一个空态面板，而是在 `snapshot.isEmpty` 时直接切换到专用页面内容区。

原因：

- 这样最接近效果图的“整页空状态”意图
- 不会被现有分区列表的节奏、留白和容器边界牵制

### 3.2 保留页面标题，不保留卡片容器

顶部仍保留 `History` 标题，作为一级页面识别锚点；标题以下整块内容切为全页空状态，不再出现 `ScreenNotePanel` 卡片容器。

### 3.3 `+` 动作直达新建事项编辑页

空状态底部圆形 `+` 按钮点击后，直接进入现有事项编辑页新建态，不通过额外确认层，也不先跳回首页。

原因：

- 这是最短的新建链路
- 与效果图中的单一主动作意图一致

## 4. 视觉约束

- 风格：温和产品感，浅色模式，克制而有留白
- 标题：大号 `History`
- 主体：插画 / 图形区域居中，不能退回成卡片式空态
- 主文案：`No history yet`
- 说明文案：强调“完成或删除的事项会出现在这里”
- `+` 按钮：圆形、单一主动作、视觉上比普通文字链接更明确
- 不引入统计块、边框卡片、额外分区头或多动作按钮

## 5. 插图策略

空状态中间插图采用“单独资源图”方案：

- 不直接复用整页效果图作为页面背景
- 不在 Flutter 里手写几何图形硬复刻插图
- 需要单独生成或导出一张只包含空状态中心插图的本地资源图
- 页面实现中由 Flutter 负责：
  - 页面标题
  - 空状态主标题
  - 说明文案
  - 底部圆形 `+` 动作
- 页面实现中由本地资源图负责：
  - 中心 clipboard / 清单 / 绿植插图区域

这样可以保证：

- 插图风格尽量贴近效果图
- 文案与交互仍保持 Flutter 原生清晰度
- 后续若要替换插图，不需要重写整页结构

## 6. 文件边界

本次优先改动：

- `lib/features/history_center/presentation/pages/history_center_page.dart`
- `lib/features/history_center/presentation/widgets/history_empty_state_panel.dart`
- `test/features/history_center/presentation/history_center_page_test.dart`

如果需要页面内直接处理跳转，可小范围补充：

- `lib/app/router/route_paths.dart` 仅用于读取已有路径常量，不新增新路由

## 7. 交互与状态

- `snapshot.isEmpty == true`
  - 显示新空状态页
  - 点击 `+` 后进入事项编辑页新建态
- `snapshot.isEmpty == false`
  - 保持当前正常态列表页面逻辑
- `loading`
  - 维持现有加载分支，不并入新空状态
- `error`
  - 维持现有错误分支，不并入新空状态

## 8. 测试要求

至少覆盖：

- 空状态时显示 `No history yet`
- 空状态时不再出现卡片式空态容器
- 空状态时显示圆形 `+` 动作
- 点击 `+` 后进入现有事项编辑页新建态
- 有历史记录时仍走原来的列表分支

## 9. 风险与控制

- 风险：空状态布局和现有滚动容器冲突，导致垂直溢出
  - 控制：空状态分支走独立布局，不依赖列表分区节奏
- 风险：`+` 动作只做样式未接真实跳转
  - 控制：直接复用现有编辑页新建路径，并补测试
- 风险：空状态改动误伤正常态列表
  - 控制：严格把新布局限制在 `snapshot.isEmpty` 分支
- 风险：插图若直接裁整页图，会把文字和留白一并带入资源，导致页面难以适配
  - 控制：改用单独插图资源图，只保留中心视觉主体

## 10. 非目标

- 不重做最近完成 / 最近删除正常态视觉
- 不修改恢复成功反馈
- 不引入新的导航模式或中间落点
