# 屏记 PRD v1.0

## 1. 文档信息

- 产品名称：屏记
- 产品定位：显示在锁屏上的极简防忘提醒工具
- PRD 版本：v1.0
- 文档日期：2026-05-22
- 平台范围：iOS 优先，Android 进入需求验证阶段
- 核心一句话：把最容易忘的小事固定在用户最常看到的地方。

## 2. 背景与机会

用户已经在用微信文件传输助手、备忘录、系统提醒事项、截图、便签等方式提醒自己，但这些方案要么容易沉底，要么入口太重，要么提醒后仍然看不见。

屏记的机会不在“管理任务”，而在“让用户一直看见那件不能忘的小事”。

技术上，iOS 已支持把应用控制项放到控制中心、锁屏和 Action Button，并可通过 WidgetKit 与 App Intents 执行动作；App Intents 也能接入 Siri、Spotlight、快捷指令、小组件和控制项。Android 侧则可通过 Jetpack Glance 构建桌面小组件，适合作为后续轻量验证方向。

参考资料：

- [Apple Developer - WidgetKit Controls](https://developer.apple.com/documentation/widgetkit/controls-collection)
- [Apple Developer - App Intents](https://developer.apple.com/documentation/appintents)
- [Apple Developer - Live Activities](https://developer.apple.com/documentation/activitykit)
- [Android Developers - Jetpack Glance](https://developer.android.com/develop/ui/compose/glance)

## 3. 产品目标

- 让用户在 3 秒内记录一件待办。
- 让重要事项在锁屏上稳定可见。
- 让过期事项不丢失、不误完成、不突然消失。
- 将早期付费点聚焦在可靠性、个性化、强提醒和多端能力上。

## 4. 非目标

- 不做复杂项目管理。
- 不做团队协作。
- 不做甘特图、看板、复杂标签系统。
- 不与 Todoist、滴答清单正面对打。
- v1.0 不追求完整 Android 客户端。

## 5. 目标用户

### 5.1 健忘型用户

经常忘记取快递、带证件、回消息、缴费等小事。对他们来说，最大痛点不是任务太多，而是记录后仍然忘记看。

### 5.2 轻量待办用户

不愿维护复杂任务系统，只想写一句话。复杂标签、项目、优先级会降低使用频率。

### 5.3 锁屏高频用户

频繁看手机，但不会主动打开待办 App。锁屏是他们最自然、最高频的提醒触点。

### 5.4 视觉提醒依赖用户

只有“看见”才会想起要做。普通通知容易被清掉，列表型 App 容易被遗忘。

## 6. 核心场景

- 出门前：锁屏显示“带身份证”“拿伞”“取快递”。
- 工作中：显示“14:00 前发合同”“回客户消息”。
- 临睡前：显示“明早 8 点带电脑充电器”。
- 临时记录：从控制中心、锁屏或快捷指令快速添加一条提醒。

## 7. MVP 范围

v1.0 只做一件事：把少量重要待办稳定显示在锁屏上，并确保不会无故消失。

### 7.1 必须包含

- 新建、编辑、完成、删除待办。
- 锁屏小组件展示。
- 过期事项继续保留。
- 最近删除 / 最近完成记录。
- 快速添加入口。
- 本地通知提醒。
- 基础隐私显示模式。

### 7.2 暂不包含

- 团队协作。
- 复杂日历视图。
- 多层级子任务。
- AI 全自动任务管理。
- 完整 Android App。

## 8. 功能需求

| 模块 | 需求 | 优先级 |

## 9. 关键用户流程

### 9.1 创建并展示事项

1. 用户打开 App。
2. 输入“明早带身份证”。
3. 选择是否设置时间、是否置顶。
4. 事项出现在锁屏小组件。
5. 用户完成后点击完成。
6. 事项进入已完成记录。
7. 如果用户没有完成，事项过期后继续显示为“已过期”，不会自动删除。

### 9.2 快速添加事项

1. 用户从控制中心、锁屏或 Action Button 触发快速添加。
2. 系统展示轻量输入界面。
3. 用户输入事项正文。
4. 默认写入“未完成事项”。
5. 锁屏小组件在系统允许的刷新窗口内更新展示。

### 9.3 恢复误删事项

1. 用户进入最近删除。
2. 找到被删除事项。
3. 点击恢复。
4. 事项回到未完成列表。
5. 操作日志记录恢复行为。

## 10. 锁屏展示规则

- 默认只展示最高优先级事项。
- 已过期但未完成事项优先级高于未来事项。
- 置顶事项优先级最高。
- 隐私模式下展示“你有 2 件事别忘了”。
- 没有事项时展示极简空状态，不制造焦虑。
- 小组件刷新失败时，保留最后一次有效内容。

## 11. 事项排序规则

排序优先级如下：

1. 置顶事项。
2. 已过期未完成事项。
3. 今天截止事项。
4. 无截止但手动置顶事项。
5. 最近创建事项。

## 13. 通知与提醒

- 普通提醒：到点发送一次通知。
- 强提醒：到点后可按用户设置重复提醒。
- 过期处理：只改变视觉状态，不自动完成、不自动删除。
- 通知点击：进入对应事项详情。
- 通知权限拒绝：App 内展示非阻断提示，引导用户改用锁屏展示。

## 14. 隐私与安全

- 锁屏正文可隐藏。
- 隐私事项默认不在锁屏展示正文。
- 删除事项进入最近删除，保留 30 天。
- 不上传用户内容到第三方模型。
- AI 提取仅作为建议，必须用户确认后创建事项。

## 15. 会员设计

### 15.1 免费版

- 基础事项管理。
- 基础锁屏小组件。
- 普通提醒。
- 最近完成记录。

### 15.2 Pro 版

- 多种锁屏样式。
- 强提醒模式。
- iCloud 同步。
- 最近删除恢复。
- 隐私展示增强。
- Apple Reminders 同步。
- AI 文本提取。
- Apple Watch / 多设备能力。

基础隐私不建议作为付费点。锁屏产品会天然暴露信息，隐私能力属于信任底座，而不是高级装饰。

- 新用户首日创建事项率不低于 60%。
- 安装后启用锁屏小组件率不低于 35%。
- 7 日留存不低于 20%。
- 任务误消失投诉为 0。
- Pro 试用或购买转化率不低于 3%。
- Android 等待名单达到 500 人后再进入开发评审。

## 19. 主要风险

- 事项消失会直接破坏信任，必须 P0 处理。
- 小组件刷新受系统限制，不能承诺实时刷新。
- 功能过多会稀释“锁屏防忘”定位。
- AI 如果自动写入事项，容易造成误提醒。
- Android 完整版过早开发会拖慢 iOS 核心体验打磨。

## 20. 验收标准

- 过期事项不会自动消失。
- 删除事项可在最近删除中恢复。
- 锁屏小组件能正确展示置顶、过期和今日事项。
- 隐私模式下锁屏不泄露正文。
- 快速添加路径不超过 3 秒。
- 未授权通知时，核心锁屏展示仍可使用。
- 所有 P0 功能无阻塞性缺陷后才允许上架或推广。

## 21. 一句话结论

屏记不要变成 Todoist，而要变成“最可靠、最顺手、最适合锁屏的一张电子便利贴”。现阶段最值得做的不是更多管理功能，而是可靠、不丢、随手记、一直看得见。
