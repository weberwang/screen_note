# Shared Prototype Playback

## Page Structure Playback

- 顶部是品牌识别与轻量状态入口，服务于“这是我的锁屏防忘中心”这一认知，不承载复杂操作。
- 首屏主区只有一个绝对主角：当前最重要事项卡片。它需要同时承载事项标题、紧急状态、时间信息与最直接的后续处理意图。
- 第二层是“紧急事项 / 当前队列”，以统一列表行承载其余待处理事项，强调可快速扫读，而不是项目管理式多维信息。
- 底部是共享壳层，只保留 `Home / History / Settings` 三个一级目的地。
- 全局快速添加独立存在，作为高频动作入口，位置稳定、足够醒目，但不抢走首要事项卡片的层级。

## Interaction Checklist

- 首页主卡片可进入事项详情或编辑链路。
- 列表行支持完成、进入详情、查看日期状态等基础交互。
- 全局快速添加可直接打开极简录入入口。
- `History` 承接最近完成与最近删除，支持恢复链路。
- `Settings` 承接通知、隐私、展示样式、同步与会员入口。
- 必须覆盖空状态、过期状态、隐私态、通知拒绝降级、Widget 刷新失败降级。
- 手机端交互优先轻量、直接、低跳转；不引入复杂多级导航。

## Please Confirm Before High-Fidelity Prototype Build

- 共享原型将以“一个主任务卡片 + 一个轻量任务队列 + 三栏共享壳层 + 全局快速添加”为固定骨架。
- 共享原型阶段只冻结共享主题、共享公共壳层、共享组件族和共享交互原则，不展开模块级最终页面实现代码。
