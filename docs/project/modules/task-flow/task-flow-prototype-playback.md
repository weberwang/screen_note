# Task Flow Prototype Playback

## 1. Scope

本次原型只覆盖 `task-flow`：

- 首页单主任务卡片
- 首页紧急队列
- 新建/编辑任务主链路
- 优先级、状态、隐私安全与备注编辑区
- 保存动作与从首页进入编辑页的主路径

本次原型不覆盖：

- `history-center` 的最近完成与最近删除恢复链路
- `settings-center` 的通知、隐私、展示模式与同步设置页
- `widget-bridge` 的稳定快照与原生 Widget 投影

## 2. Page Structure Playback

### task-flow-home

1. 顶部先给出轻品牌识别与问候语，不承载复杂操作。
2. 第一阅读层始终是单个主任务卡片，承载标题、简述、到期状态和直接进入编辑的动作。
3. 第二阅读层是紧急队列，使用轻行式列表承接其余待处理事项，不回退成高密度任务面板。
4. 共享三栏壳层与全局快速新增继续可见，但视觉优先级低于主任务卡片。

### task-flow-editor

1. 顶部是取消与轻操作入口，中间是当前页面标题与简短说明。
2. 首个主卡集中承载标题、到期时间、优先级、状态、隐私安全和备注编辑。
3. 附件、提醒等次级支持项下沉到主卡之后，不抢首屏编辑路径。
4. 底部单独保留主保存动作，形成清晰的“编辑后立即保存”闭环。

## 3. Interaction Checklist

- 点击首页主任务卡片或右下箭头，可进入当前优先任务的编辑页。
- 点击紧急队列任一行，可带着对应任务上下文进入编辑页。
- 点击全局快速新增，可进入新建态编辑页。
- 编辑页支持切换优先级与状态的单选态，验证视觉层级与交互反馈节奏。
- 编辑页支持切换隐私安全开关，验证“可见内容与安全展示”边界。
- 编辑页保存后回到首页，模拟最小闭环，不扩展真实持久化逻辑。

## 4. Fidelity Notes

- 首页必须保持“一个主任务 + 一个轻队列”的清晰层级，不得压缩成多卡片堆叠。
- 主任务卡片允许轻纸感与暖色表面，但不能漂移成强便签隐喻。
- 队列必须保持行式结构，避免把次级任务抬升成与主任务同级的大卡片。
- 编辑页必须先暴露核心任务字段，再下沉次级配置，不得做成设置中心式密集表单。
- 全局快速新增继续是共享高频动作，但不得盖过主任务标题的首要识别度。
